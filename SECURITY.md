# Security Policy

## Supported Versions

Since this project is primarily a documentation repository and an educational guide, "versions" refer to the major updates of the guide itself. We strive to keep the guide updated with the latest LTS (Long Term Support) release of Ubuntu.

| Version | Supported          | Ubuntu LTS Version |
| ------- | ------------------ | ------------------ |
| v1.0.x  | :white_check_mark: | 24.04 LTS          |
| v0.9.x  | :x:                | 22.04 LTS          |

## Reporting a Vulnerability

If you discover a security vulnerability in the shell scripts (`scripts/`) or find a step in the documentation that suggests insecure practices (e.g., advising users to use `chmod 777` or disable firewalls incorrectly), please let us know immediately.

1. Do **NOT** open a public issue.
2. Email the project maintainers directly at `security@example.com` (replace with actual contact in production).
3. Provide a detailed description of the vulnerability or insecure practice.
4. If applicable, suggest a secure alternative or fix.

We will acknowledge receipt of your vulnerability report within 48 hours and strive to resolve it quickly. Public disclosure should only happen after the issue has been patched or the documentation updated.
